## Context

Implement **Screen 1** of the NASI onboarding flow: the welcome + mode picker.

**FigJam source**: `https://www.figma.com/board/O9ONqUiEu0vcaAJ5VGgq1d/NASI?node-id=39-825`
**FigJam node**: Section `3:3909` — "Screen 1"
**Experiment dir**: `experiments/001-nasi-mode-picker/`
**Page route**: `src/pages/experiments/001-nasi-mode-picker/index.astro` → `/experiments/001-nasi-mode-picker/`

### What the FigJam shows

A single-screen mobile layout (target ~390px wide) with:

1. **Welcome card** (top, large rounded rect) — copy reads:
   > "hello message
   > lets pick a vibe
   > you can always toggle between modes"

   Designer note: animated on load — pops in first, then minimizes to give space for the mode cards below.

2. **Three mode cards** (tall rounded rects, stacked vertically):
   - **Loom** — blue
   - **Sprite** — pink/coral
   - **Shell** — gray

Each card is a tap target that selects the mode and advances to Screen 2.

### What to build

A React component `NasiModePicker.tsx` in `experiments/001-nasi-mode-picker/src/`. Mobile-width layout (~390px), centered on the page. The welcome card should have an entry animation (CSS or Framer if available, otherwise Tailwind `animate-`). The three mode cards should have distinct background colors matching the mode identity.

No routing logic needed — this is a static visual. Cards can log to console on click.

## Acceptance Criteria

- [ ] Layout matches the FigJam structure: welcome card at top, three mode cards stacked below
- [ ] Loom = blue, Sprite = pink/coral, Shell = gray — consistent with the color system used in all subsequent screens
- [ ] Welcome card has a visible entry animation
- [ ] Component renders correctly at 390px viewport width
- [ ] Page at `/experiments/001-nasi-mode-picker/` displays the component
