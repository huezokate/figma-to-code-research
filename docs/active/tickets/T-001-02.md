## Context

Implement **Screen 2** of the NASI onboarding flow: the app permissions / data connect screen.

**FigJam source**: `https://www.figma.com/board/O9ONqUiEu0vcaAJ5VGgq1d/NASI?node-id=39-825`
**FigJam node**: Section `3:3935` — "Screen 2"
**Experiment dir**: `experiments/002-nasi-app-connect/`
**Page route**: `src/pages/experiments/002-nasi-app-connect/index.astro` → `/experiments/002-nasi-app-connect/`

### What the FigJam shows

**Tab bar** (top): Three tabs — Sprite (selected/active), plus two empty tabs. Tab bar is shared across screens 2–6. Sprite is pink/coral when selected.

**Heading**: "Let's get you set up and connected"

**Body copy**: "Select the apps you would like give Sprite access to (include those for Loom and Shell?)"

**App grid** (3 columns, scrollable):
Row 1: Apple Music, iMessage, Instagram
Row 2: Spotify, Podcast, Pinterest
Row 3: X (twitter), Wattpad, TikTok
Row 4: Oura, Picture Gallery, YouTube
Row 5: Disney (partial row)

Each app is a rounded square tile with label. Tappable (toggle selected/unselected state).

**Security bar** (bottom, pinned): Dark background with text — "Ideally a super message on security and never selling or using data for evil?" (lo-fi placeholder for real security copy).

### What to build

A React component `NasiAppConnect.tsx` in `experiments/002-nasi-app-connect/src/`. Includes:
- `ModeTabBar` sub-component (Sprite/Shell/Loom tabs) — extract this as a separate file since it's reused in screens 3–6
- `AppTile` sub-component (icon placeholder + label, toggleable selected state)
- App grid with 3-column layout
- Pinned security footer

Mobile width (~390px). App tiles use emoji or a letter placeholder as the icon (no real icons needed for PoC).

- [ ] Security footer is pinned to bottom of screen
